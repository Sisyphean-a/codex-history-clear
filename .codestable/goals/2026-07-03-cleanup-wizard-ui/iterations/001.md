# Iteration 001

## Current Understanding

本轮不是补后端链路，而是把已经可用的扫描、计划、备份、删除、回滚、证据能力，重新组织成普通用户看得懂的“安全清理向导”。验收重点除了界面结构，还包括主流程不能再泄露工程细节，会话范围不能失真，自动备份复选项必须是真能力。

## Implementation Approach

- 保留后端历史删除链路，只补目录切换、会话大小和自动备份开关所需的最小字段与绑定。
- 把前端控制器拆成 store / actions / builders / view，压低文件和函数复杂度。
- 主流程只保留扫描、策略、筛选、列表、计划和确认；discovery、manifest、恢复记录、校验报告全部收进高级详情。
- 用独立子代理做功能验收，不拿测试结果代替产品验收。

## Changes This Iteration

- 重写主界面信息架构，做成“顶部概览 + 左侧策略栏 + 右侧会话列表/执行计划 + 高级详情折叠区”。
- 会话列表增加大小、项目目录、状态标签、手动选择与策略选择，并支持按时间、归档状态、项目目录、大小筛选。
- 目录切换支持自定义 `.codex` 根目录；扫描统计支持会话大小；会话读取改成不限量，避免 400 条硬上限导致统计失真。
- 删除确认口令改成 `CLEAN`；“只备份不删除”不再被口令卡住。
- “删除前自动备份”从假开关改成真能力：前端把选择传给后端，后端据此决定是否生成备份和恢复记录。
- 主流程中的 `journal`、`rollback`、`evidence` 工程词改成人话；相关文件路径和调试信息只留在高级详情。

# Iteration 005

## Current Understanding

第 2 步 discovery 枚举已经稳定后，当前的关键不是继续扩范围，而是把只读扫描真正闭环成 artifact：`discovery.json`、`manifest-before.json` 和 `unknown-items.json` 必须真实落盘，桌面工作区也要投影这些结果，而不是继续停留在占位路径。

## Implementation Approach

在 `internal/discovery` 里补 artifact writer 和 manifest 基线投影；把 `content_hash`、`storage_kind` 和 unknown-items 的最小稳定 schema 一次写正；然后用 fixture 测试、前端构建、Wails 打包和一次真实扫描命令验证落盘结果。

## Changes This Iteration

- discovery 主流程从“只返回路径字符串”推进到“真实写 artifact”：
  - `RunReadOnlyScan` 现在会创建输出目录并落盘 `discovery.json`、`manifest-before.json`、`unknown-items.json`
  - `codex doctor --json` 在返回有效 JSON 时会额外写 `codex-doctor.json`，并把相对路径写入 `discovery.json`
- 新增 manifest 基线投影：
  - `history_jsonl` → `codex_history_jsonl`
  - `rollout_jsonl` / `archived_rollout_jsonl` → `codex_rollout_jsonl`
  - `state_sqlite` / `logs_sqlite` → `codex_sqlite`
  - 为基线记录补 `canonical_path`、`real_path`、`reparse_kind`、`content_hash`、`evidence`
  - `auth_json`、`config_toml`、`credentials_json`、`session_index_jsonl` 进入 `unknown-items.json`
- 前端结果面板增强：
  - 结果区明确展示 CLI 可用性、doctor 状态、resume 支持、artifact 路径和前 5 个对象预览
  - warning 和无 warning 状态分开展示
- 测试升级为真实落盘验证：
  - fixture 单测确认 artifact 文件存在、内容可解析、manifest 记录数正确、输出路径未逃逸 `OutputDir`

# Iteration 010

## Current Understanding

当前真正缺的不是“再补一点 UI”或“再猜一下数据结构”，而是把 Codex 当前本地模型下的真实历史删除链路完整做通：`state_5.sqlite.threads` 负责会话主索引，`logs_2.sqlite`、`goals_1.sqlite`、`session_index.jsonl`、`history.jsonl`、`.codex-global-state.json(.bak)` 和 rollout 文件共同组成真实删除面。

## Implementation Approach

先按参考项目和本机真实 `.codex` 结构重建 destructive 执行链路：列表、计划、备份、改写、回滚、复扫、执行后校验一个都不缺；再把桌面端从“只读预览”扩成“可筛选、可确认、可执行、可回滚”的真实工作区；最后用夹具测试和整包构建证明闭环成立。

## Changes This Iteration

- 新增 `internal/history/` 服务层，落地真实历史删除闭环：
  - 从 `state_5.sqlite.threads` 列出本地会话，并回填 `session_index.jsonl` 标题
  - 生成真实删除计划，覆盖 `state/logs/goals/session_index/history/global_state/rollout/shell snapshot`
  - destructive 前先备份原文件并生成 `rollback-journal.json`
  - 执行 SQLite / JSONL / JSON / rollout 改写后，生成 `approved-plan.json`、`exec-result.json`、`manifest-after.json`
  - 执行后做一致性校验；失败时自动恢复备份
- 新增桌面绑定：
  - `ListHistoryThreads`
  - `BuildHistoryDeletePlan`
  - `ExecuteHistoryDeletePlan`
  - `RollbackHistoryDelete`
- 新增桌面历史工作区：
  - 会话筛选
  - 选择目标并生成计划
  - 输入 `purge-selected` 确认真实删除
  - 展示 rollback / exec-result / manifest-after / verification
  - 支持按 journal 回滚
- 补充夹具测试，覆盖列表、计划、真实删除、回滚。
- 额外做了一轮结构收敛：拆超长函数、拆超长测试文件、拆前端历史工作区和样式文件，让本轮新增代码满足仓库自己的尺寸约束。

## Problems Encountered

- Windows 下 JSONL 原子替换一开始被占用句柄卡住，已改成先关闭源句柄再替换。
- SQLite 校验阶段一开始因为单连接下嵌套查询触发死锁，已改成先收表名、再逐表查列。
- TypeScript 编译期要求在异步闭包里重新收窄 `planResult` / `executionResult`，否则不放行。
- `modernc.org/sqlite` 引入后，`go mod tidy` 会把 `go.mod` 语言版本稳定写回 `1.25.0`；当前这不是脏改，而是依赖图的最终结果。
- 本轮尝试启动独立 Task agent 功能验收，但平台在本 turn 内没有返回 auditor 结论，因此还不能把 goal 标成 complete。

# Iteration 011

## Current Understanding

上一轮独立 auditor 已经把 goal 没法关闭的原因讲清楚了：不是主链路不存在，而是最后几块验收门还没补完，包括后端硬审批、`backupOnly` 死字段、`go test ./smoke/...` 缺失、evidence pack 导出缺失，以及缺少可见的桌面运行证据。

## Implementation Approach

这轮不再扩需求，只定点补 auditor 点名的阻塞项：把批准动作从前端口令改成后端 `approved plan` gate；让 `backupOnly` 真能返回预演结果；补 smoke 目录和最小闭环；补 evidence pack 导出；最后用真实启动截图和 fresh 构建结果重新喂给独立 auditor。

## Changes This Iteration

- 后端真实删除执行链补了两层门：
  - 新增 `ApproveDeletePlan` / `ApproveHistoryDeletePlan`，先把 `delete-plan.json` 显式转成 `approved-plan.json`
  - `ExecuteDeletePlan` 现在只接受 `approved=true` 的计划文件，不再允许“前端直接传 `confirmed=true` 就删”
- `backupOnly` 现在不再是死字段：
  - 可生成备份、rollback journal、exec result
  - 不做 destructive 改写
  - 返回 `mode=backup-only` 与 `verification.status=skipped`
- 新增 evidence pack 导出：
  - 后端可把 `delete-plan / approved-plan / rollback-journal / exec-result / manifest-after` 组织成 `evidence-pack.json`
  - 前端执行结果面板新增导出入口
- 新增 smoke 闭环：
  - 新建 `smoke/` 测试包
  - `go test ./smoke/...` 现在覆盖一条成功路径和一条未批准计划错误路径
  - 新建 `fixtures/smoke/README.md` 说明 smoke 夹具约定
- 前端历史工作区补强：
  - 新增“只做备份”入口
  - 执行结果显示 mode、verification summary、残留引用明细和事件流
  - Evidence Pack 导出路径可见
- 收掉桌面旧产物名：
  - `wails.json.outputfilename` 改成 `codex-history-manager`
- 补桌面可见证据：
  - `.codestable/goals/2026-06-30-codex-history-manager/artifacts/workspace-shell.png`
  - `.codestable/goals/2026-06-30-codex-history-manager/artifacts/desktop-startup.png`
  - `.codestable/goals/2026-06-30-codex-history-manager/artifacts/desktop-startup-screen.png`
- 同步 requirement / ADR / roadmap / checklist 口径，清掉“默认不直改 live SQLite/JSONL”“roadmap 仍 in-progress”“checklist 全 pending”这些和当前实现冲突的旧状态。

## Problems Encountered

- Wails 桌面窗口的 WebView 内容无法通过 `PrintWindow` 稳定抓到，所以 `desktop-startup.png` 只能证明窗口确实启动，不能完整证明页面内容。
- in-app Browser 插件在当前环境里不可用（`agent.browsers.list() == []`），因此改用本机 Edge headless 对静态前端壳层截图，补齐工作区可见证据。
- 旧 requirement / roadmap / checklist 大量仍停留在“只读/待实现”口径，如果不先对齐，独立 auditor 会继续把这些文档冲突当成 goal 未完成证据。

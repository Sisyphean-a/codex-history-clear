# Iteration 004

## Current Understanding

第 1 步 Wails2 壳层和扫描工作区已经能稳定启动，第 2 步需要把 discovery 从 stub 结果推进到真实的候选根目录枚举、对象分类和 CLI 诊断采集，同时继续保持只读边界。

## Implementation Approach

在 `internal/discovery` 拆出根目录解析、文件分类、CLI 探测三个子模块；用 temp fixture 驱动单测锁住对象清单、CLI warning 和非法根目录报错，再把新增字段通过 Wails 绑定带回桌面壳层。

## Changes This Iteration

- 将 discovery 层从 stub 预演改为真实只读发现：
  - 支持 `CODEX_HOME` / 用户目录 `.codex` / extra roots 的候选根目录解析与去重
  - 递归识别 `config_toml`、`auth_json`、`credentials_json`、`history_jsonl`、`session_index_jsonl`、`state_sqlite`、`logs_sqlite`、`rollout_jsonl`、`archived_rollout_jsonl`
  - 返回对象级元数据：源根目录、绝对路径、大小、UTC mtime、基础属性、link 元数据
- 新增 CLI 诊断快照：
  - `codex` 不存在时进入 warning
  - `codex --help` 推断 `resume_supported`
  - `codex doctor --json` 失败或返回无效 JSON 时进入 warning
- 扩展 Wails 绑定返回面：
  - `ScanResult` 新增 `Items` 和 `CLISnapshot`
  - 前端工作区文案从 `stub scan` 改为真实 discovery 语义，并展示 `itemCount`
- 重写 `internal/discovery/service_test.go`：
  - fixture 成功枚举场景
  - CLI 缺失 warning 场景
  - 不存在根目录显式报错场景

# 筛选结果无法全选 Issue Report

## 1. 问题现象

会话列表可以筛选出一个月之前的会话，但无法一次选中全部筛选结果，只能逐条勾选。

## 2. 复现步骤

1. 进入会话清理页面并完成扫描。
2. 将时间筛选设为“一个月之前”。
3. 尝试通过列表表头的勾选框选中当前筛选结果。
4. 观察到：筛选结果没有被全部选中，只能逐条勾选会话。

复现频率：稳定。

## 3. 期望 vs 实际

**期望行为**：筛选出一个月之前的会话后，可以一次选中当前全部筛选结果，再进行批量删除。

**实际行为**：筛选功能有效，但没有可用的全选操作，必须逐条勾选。

## 4. 环境信息

- 涉及模块 / 功能：会话清理页的时间筛选、列表选择和批量删除
- 相关文件 / 函数：前端会话列表与选择状态
- 运行环境：当前项目本地环境
- 其他上下文：未发现已有同类 issue；当前工作区无相关未提交改动

## 5. 严重程度

**P1** — 批量清理是核心操作；现有逐条勾选可以绕过，但会话较多时操作成本很高。

## 备注

用户希望全选只作用于当前筛选结果，例如“一个月之前”的所有会话。
